# Project Context

## オリジナルの要望（Shunより）

以下はプロジェクト開始時に Shun が言語化した要望の要約：

### 中核目的
- 毎日5〜10分で読める（または聞ける）ニュースキュレーション
- バイアスバランスの取れた情報源（欧米・アジア・中東・参照として中露）
- **半年〜1年先を予測するための材料を蓄積したい**
- 単発の要約ではなく、**長期的に何が変わったかを理解したい**

### 関心領域
- テック（特にAI）
- 政治・地政学
- 経済
- スタートアップ（資金調達・新興企業・プロジェクト）
- AIで今後何が可能になるか（将来予測示唆）

### 排除したいもの
- 関心のない分野のノイズ
- 一方向のバイアス（特に状態メディアを中立扱いすること）

### 配信形式
- Webアプリ + メール両方
- 英語原文ベース

### 長期ビジョン（Shunの言葉ベース）
> 「自律的に新しくニュースが変わったことによって、過去の情報と今日の情報で何が影響を受けて
> 今後何が予測できるのかっていうことを考えられるような、どんどん自律的に改善させていく
> ようなシステム」

## バイアスバランスへの考え方

Shun の懸念：
> 「共産主義とかのニュースは政府情報に偏ってしまいがちなので、中国とかロシアとか
> どうやってバランス取ればいいのか分かんないんだけど、そこもある程度少し入れて
> バランスを取れるような情報を収集したい」

我々の対応方針：
- 国営メディア（Global Times, RT, Xinhua）は **`source_region: "State Media"` でタグ付け**
- 「○○政府の公式見解」として位置付け、独立した報道とは区別
- 中国・ロシアに関する話題は、SCMP（香港）や独立系アジアメディアでクロスチェック
- 各日次ブリーフの `bias_note` で「今日はどの地域の視点が不足したか」を明記

## 主要設計判断とその根拠

### 判断1: GitHub Actionsをバックエンドに
- **代替案**: ローカルcron, VPS, Cloudflare Workers
- **採用理由**: 無料、Gitで履歴管理、APIキーをsecretsで安全に管理、メンテ不要
- **トレードオフ**: 6分制限の中で全カテゴリ処理する必要あり → カテゴリごとに並列ジョブで対応

### 判断2: SQLite + JSONエクスポート
- **代替案**: 純粋なJSONファイル、Supabase等のクラウドDB
- **採用理由**:
  - トレンド分析（過去N日間の頻度・推移）にSQLが圧倒的に楽
  - SQLiteは1ファイルでバックアップ簡単・Python標準搭載
  - JSONエクスポートでGitに人間可読な履歴を残せる（diffも見られる）
- **トレードオフ**: SQLite本体はバイナリなので、Git管理はJSONエクスポート側

### 判断3: AIを独立カテゴリにせずTechに統合
- **Shunの明示的指定**: 「AIをテックに統合して、代わりに『AI予測示唆』を独立セクションに」
- **意図**: AIのニュースはTechの一部だが、「AIで何が可能になるか」という未来予測は別物として扱いたい

### 判断4: Claude API + web_search ツール
- **代替案**: NewsAPI + Claude, RSS + Claude, 複数LLM
- **採用理由**: web_searchツールでリアルタイム情報が取れる、単一プロバイダで完結
- **コスト試算**: 1カテゴリあたり約$0.10 × 5カテゴリ × 30日 = 月$15程度
