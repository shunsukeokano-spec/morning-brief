# Claude Code 開発指示書

このプロジェクトは Shun の個人ニュースキュレーション・トレンド分析システムです。
このファイルは Claude Code 起動時に自動的に読まれます。

## プロジェクトの目的

Shun は世界の動向（テック・政治・経済・スタートアップ・AI）を毎日5〜10分で把握し、
**半年〜1年先の動きを予測する材料**として情報を蓄積したい。
単発のニュース要約ではなく、**長期トレンドの追跡と予測示唆の生成**が本質的価値。
日々のフィードバックを通じて常にニュースフィードを改善し続ける

## ユーザー Shun について

- 生成AIの利用に関心あり。非エンジニアがビジネスなどで活用した先端事例を知りたい
- これまで生成AIを活用して、Raspberry Pi を使った趣味工作の経験あり
- 日本語・英語両方読める。ニュース原文は英語のままでOK
- 翻訳が必要な場合: 固有名詞・公式名称は必ずウェブ検索で確認、自然な英語/日本語に

## 設計の中核判断（変更時は要相談）

| 判断 | 選択 | 理由 |
|------|------|------|
| バックエンド実行環境 | GitHub Actions | 無料・cron・履歴がGitに残る |
| データ蓄積 | SQLite + JSONエクスポート | トレンド分析にSQLが楽、JSONはGit管理用 |
| AI関連の扱い | Techに統合 + 独立「AI Forecast」セクション | Shunの明示的指定 |
| ニュース言語 | 英語原文 | Shunの指定。要約も英語ベース |
| 配信形式 | Webアプリ + メール両方 | Shunの指定 |
| LLM | Claude（Anthropic API + web_search ツール） | 単一プロバイダで完結 |

## バイアスバランス方針

各カテゴリで以下の地域から情報を収集する：
- **Western**: Reuters, AP, BBC, FT, NYT
- **Asia**: Nikkei Asia, SCMP, NHK World, The Hindu
- **Middle East / Global South**: Al Jazeera
- **State Media (参照のみ)**: Global Times, RT — 必ず `source_region: "State Media"` でタグ付け

カバレッジが偏った日は `bias_note` フィールドに明記すること。

## コーディング規約

- **Python**: 3.11+, type hints 必須, `ruff` でフォーマット
- **依存関係は最小化**: 標準ライブラリで済むなら使わない
- **環境変数**: `ANTHROPIC_API_KEY` のみ必須。それ以外は `config.toml`
- **ログ**: `logging` モジュール、INFO レベルがデフォルト
- **エラーハンドリング**: ニュース1件の失敗で全体を止めない（部分的失敗を許容）

## 開発の進め方

新しい機能を追加する前に必ず：

1. `docs/ROADMAP.md` を読んで次のフェーズを確認
2. 変更が中核判断に関わる場合は Shun に確認
3. SQLite スキーマを変更する場合は `backend/db.py` のマイグレーションを追加
4. プロンプトの変更は `backend/prompts.py` で集中管理し、変更履歴をコミットメッセージに残す

## 継続的改善の中核設計

**必ず読むこと**: [docs/IMPROVEMENT.md](docs/IMPROVEMENT.md)

このシステムの本質は「自律的に改善する」こと。改善層の設計判断：

| 要素 | 採用 | 不採用 |
|------|------|--------|
| 明示フィードバック | コールドスタート14日のみ | 恒久的な評価ボタン |
| トレンド検出 | 収集段階のクラスタリング | 事後の関連性付与 |
| プロンプト改善 | 月次自動適用 + ロールバック | 手動レビュー |
| ストーリー選択 | 80% パーソナライズ + 15% トレンド + 5% Serendipity | 純粋なランキング |
| 日曜 Serendipity Day | 30% 比率に引き上げ | オプトアウト不可 |

### 自動プロンプト改善のセーフガード（厳守）

- 改善は月1回のみ
- `prompts_history` テーブルに全変更を記録
- 適用後7日間でメトリクス（クリック率・滞在時間）が20%以上悪化したら自動ロールバック
- ロールバック時は Shun にメール通知

## 長期ロードマップ

1. **Phase 1**: 毎日収集 + 蓄積（クラスタリングなし）
2. **Phase 2**: シグナル収集インフラ + コールドスタート UI + メール・Webデプロイ
3. **Phase 3**: クラスタリング + 80/15/5 ストーリー選択
4. **Phase 4**: 嗜好モデル（EMA）+ 月次自己評価
5. **Phase 5**: プロンプト自動改善 + セーフガード
6. **Phase 6**: 音声配信（優先度低）

## やってはいけないこと

- 中核判断（上記の表）を相談なしに変更しない
- 中国・ロシアの国営メディアを「中立的ソース」として扱わない（必ず参照タグ付け）
- ニュースの政治的色付けを Claude が勝手にしない（要約は事実ベース、解釈は別フィールド）
- API キーや個人情報をコミットしない（`.gitignore` を厳格に）
- **嗜好モデルの「興味なし」シグナルだけでカテゴリ全体を遮断しない**（フィルターバブル防止）
- **Serendipity Day（日曜の多様性注入）をオプトアウト可能にしない**
- **プロンプト自動改善のロールバック機能を省略しない**
- **コールドスタート期間（14日）を「絶対」にしない**（Shun が UI を早く消したければ消せる）
